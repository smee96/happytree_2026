# HappyTree 시뮬레이션 규칙

## 📋 핵심 규칙

### 1. 입장 및 초기화
- **환영 하트**: 신규 입장 시 5개 지급
- **초기 하트허용치**: 0
- **초기 레벨**: 0 (레벨업 전)

### 2. 하트허용치 증가 규칙
- 새로운 사용자 1명 입장 시, **모든 선발 입장자**의 하트허용치 +1
- 예시: 8번째 사용자 입장 시, 1~7번 사용자 모두 하트허용치 +1

### 3. 레벨업 조건 (모두 충족해야 함)
1. **하트허용치** ≥ hearts_required
2. **보유 하트** ≥ hearts_required
3. **별 자동 구매**: 필요한 별은 자동으로 구매 (조건 아님)

### 4. 레벨업 시 처리
1. **별 구매**: stars 개수만큼 자동 구매 ($2/개)
2. **하트 소비**: 
   - 보유 하트 차감: -hearts_required
   - 하트허용치 차감: -hearts_required
3. **보상 획득**:
   - 하트 보상: +hearts_reward
   - 코인 보상: +coins ($0.10/개)

### 5. 레벨업 자동화
- 조건 충족 시 **즉시 자동 레벨업** (대기 없음)
- 매 입장 후 모든 사용자의 레벨업 재시도

## 💰 화폐 체계
- **별(★)**: $2/개 (사용자 구매)
- **코인(◎)**: $0.10/개 (플랫폼 지급)
- **하트(♥)**: 무료 (레벨업 보상으로만 획득)

---

## 📊 통계 산출 방법

### 레벨별 달성자 수
- **1개 화분 모드**: 해당 레벨을 통과한 사용자 수
  - 예: Lv.3 달성자 = currentFarmIndex > 2인 사용자 수

### 플랫폼 수익 계산
1. **총 별 판매** = Σ(레벨별 달성자 수 × 필요 별)
2. **별 수익** = 총 별 판매 × $2
3. **코인 지출** = Σ(레벨별 달성자 수 × 보상 코인) × $0.10
4. **순수익** = 별 수익 - 코인 지출
5. **수익률** = (순수익 / 별 수익) × 100%

### 사용자 수익 계산
1. **투자금** = 구매 별 × $2
2. **수익금** = 획득 코인 × $0.10
3. **순이익** = 수익금 - 투자금
4. **ROI** = (순이익 / 투자금) × 100%

## 🎯 검증 예시

### 8명 입장 시나리오
**1번 사용자 진행 과정:**

1. **초기 상태**: 하트 5개, 허용치 0
2. **2번 입장**: 허용치 1 → Lv.1 달성 (하트 5→5, 허용치 1→0)
3. **3번 입장**: 허용치 1
4. **4번 입장**: 허용치 2
5. **5번 입장**: 허용치 3 → Lv.2 달성 (하트 5→6, 허용치 3→1)
6. **6~8번 입장**: 허용치 4

**최종 결과:**
- 레벨: 농장 1 - Lv.2
- 보유 하트: 6개
- 하트허용치: 4
- 구매 별: 0개
- ROI: 0.00%

## ⚠️ 중요 참고사항

1. **cum 필드**: 문서 참고용, 레벨업 조건에는 hearts_required만 사용
2. **자동 레벨업**: 조건 충족 시 즉시 처리 (선택 불가)
3. **별 자동 구매**: 레벨업 조건에 별이 필요하면 자동 구매
4. **Lv.8 보상**: 하트 보상 없음 (최종 레벨)
5. **256번 사용자**: 농장 2 시작 위치 (총 입장자 ≥ 256일 때만 표시)
