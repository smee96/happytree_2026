# HappyTree - 농장별 시뮬레이터

## 프로젝트 개요
- **이름**: HappyTree - 농장별 독립 시뮬레이터
- **목표**: 4개 농장을 독립적으로 운영하여 각 농장의 경제 메커니즘과 수익성을 검증
- **주요 기능**:
  - 농장별 독립 시뮬레이션 (각 농장마다 별도 입장인원/화분수/별가격 설정)
  - 하트허용치는 같은 농장 내에서만 증가
  - 농장별 플랫폼 수익 및 사용자 ROI 분석
  - 1~5개 화분 확장 지원
  - $1~$5 별 가격 유연 설정
  - 실시간 농장별 비교 분석

## API 엔드포인트

### 농장별 독립 계산 API (신규 - 권장)
- `GET /api/farm/:farm_id/:total_users/:max_pots?starPrice=2`
  - **farm_id**: 1~4 (농장 번호)
  - **total_users**: 1~1,000,000 (입장 인원)
  - **max_pots**: 1~10 (화분 개수)
  - **starPrice**: 1~5 (별 가격, 기본값 $2)

**사용 예시:**
```bash
# 농장 1, 100명, 3화분, 별 $2
curl https://happy-tree.pages.dev/api/farm/1/100/3?starPrice=2

# 농장 2, 1500명, 5화분, 별 $3
curl https://happy-tree.pages.dev/api/farm/2/1500/5?starPrice=3
```

## 게임 메커니즘

### 화폐 체계
- **별(★)**: $1~$5/개 (유연 설정), 사용자 구매, 레벨업 필수
- **코인(◎)**: $0.10/개, 레벨 완료 시 보상, 현금화 가능
- **하트(♥)**: 무료, 레벨업 투입, 현금화 불가

### 하트허용치 메커니즘 (핵심 변경)
- **정의**: 같은 농장 내 후발 입장자가 만드는 화분 개수의 합계
- **계산**: n번 사용자의 하트허용치 = ∑(n+1번~마지막 사용자의 화분 개수)
- **농장 독립**: 농장 1 신규 입장 → 농장 1의 선발 사용자에게만 하트허용치 증가
- **특징**: 선발자일수록 유리, 화분 개수가 많을수록 후발자 기여도 증가

### 레벨업 조건 (AND)
1. 보유 별 ≥ 레벨별 필요 별
2. **하트허용치 ≥ 레벨별 필요 하트허용치** (농장별 독립)
3. 보유 하트 ≥ 레벨별 필요 하트

## 핵심 개선사항

### 1. 농장별 독립 운영
- 각 농장은 완전히 독립적으로 운영
- 하트허용치는 같은 농장 내에서만 증가
- 농장 간 영향 없음

### 2. Lv1-3 진입장벽 제거
- Lv1-3의 하트허용치를 0으로 설정
- 초기 무료 진입 가능
- Lv4부터 본격 투자 시작

### 3. 코인 보상 50% 감소
- 농장 2/3/4의 코인 보상을 50% 감소
- 플랫폼 수익성 대폭 개선
- 흑자 전환 달성

### 4. 화분 개수에 따른 하트허용치 증가
- 기존: 후발 입장자 수만큼 증가
- 변경: 후발 입장자가 만드는 화분 개수 합계만큼 증가
- 효과: 화분 확장 시 선발자에게 더 큰 혜택

## 사용 방법

### 웹 UI 사용
1. https://happy-tree.pages.dev 접속
2. 원하는 농장 탭 선택 (🌱1, 🌿2, 🌳3, 🌲4)

## 핵심 통찰

### 농장별 특징
- **농장 1**: 진입 쉬움, 빠른 ROI 회수, 낮은 수익률
- **농장 2**: 중간 난이도, 높은 하트허용치 요구

### 수익성 분석
- **플랫폼**: 농장 2/3/4가 높은 수익률 (95%~100%)
- **사용자**: 농장 1이 상대적으로 유리 (ROI -47.50%)
- **별 가격**: 높을수록 플랫폼 수익 증가, 사용자는 동일

### 지속 가능성
- **하트허용치 의존**: 후발 입장자 필수
- **화분 확장 효과**: 화분이 많을수록 선발자 유리
