# HappyTree 시뮬레이션 엔진

## 프로젝트 개요
- **이름**: HappyTree 시뮬레이션 엔진
- **목표**: 작물 키우기 게임의 경제 메커니즘을 시뮬레이션하여 사용자 수익률과 플랫폼 수익성을 검증
- **주요 기능**:
  - 다양한 시나리오 시뮬레이션 (일일 입장자 수, 시뮬레이션 기간 설정)
  - 하트허용치 메커니즘 자동 계산
  - 4개 농장, 각 8레벨 자동 진행
  - 사용자별 투자 수익률(ROI) 분석
  - 플랫폼 수익성 분석
  - 실시간 차트 및 통계 시각화

## API 엔드포인트

### 🚀 핵심 계산 API (권장)
- `GET /api/calculate/:total_users` - **입력 인원수에 따른 핵심 지표 즉시 계산**
  - **1번 사용자 상태값** (최고 우선순위, 가장 많은 하트허용치)
  - **중간 사용자 상태값** (입장 순서 N/2)
  - **마지막 입장자 상태값** (가장 불리한 위치)
  - **플랫폼 수익** (총 별 판매, 총 코인 지급, 순수익)
  - ✨ **초고속 응답** (DB 없이 순수 계산, 1500명도 0.2초 이내)

**사용 예시:**
```bash
# 255명 (Farm 1 Lv8 완료 최소 인원)
curl http://localhost:3000/api/calculate/255

# 1500명 (대규모 시나리오)
curl http://localhost:3000/api/calculate/1500
```

### 시뮬레이션 세션 (전체 DB 시뮬레이션)
- `POST /api/sessions` - 새 시뮬레이션 세션 생성
- `GET /api/sessions` - 모든 세션 조회
- `GET /api/sessions/:id` - 특정 세션 조회
- `POST /api/sessions/:id/run` - 시뮬레이션 실행 (느림, 대규모는 권장하지 않음)

### 분석
- `GET /api/sessions/:id/users` - 세션의 모든 사용자 조회
- `GET /api/sessions/:id/analysis` - 사용자별 투자 수익률 분석
- `GET /api/sessions/:id/stats` - 플랫폼 통계 조회

### 농장 설정
- `GET /api/farms` - 모든 농장 설정 조회
- `GET /api/farms/:farmId/levels` - 농장별 레벨 설정 조회

## 게임 메커니즘

### 화폐 체계
- **별(★)**: $3/개, 사용자 구매, 레벨업 필수
- **코인(◎)**: $0.10/개, 레벨 완료 시 보상, 현금화 가능
- **하트(♥)**: 무료, 레벨업 투입, 현금화 불가

### 하트허용치 메커니즘
- **정의**: 특정 사용자의 하트허용치 = 그보다 뒤에 입장한 사람 수 + 1
- **공식**: n번 사용자의 Day d 하트허용치 = (일일 입장 수 × d) - n + 1
- **특징**: 선발자일수록 유리, 후발자는 신규 입장자가 계속 필요

### 레벨업 조건 (AND)
1. 보유 별 ≥ 레벨별 필요 별
2. 하트허용치 ≥ 레벨별 필요 하트허용치
3. 보유 하트 ≥ 레벨별 필요 하트

### 농장 구조
- **농장 1**: 기본 개방, 총 6별 투자, 186코인 수익
- **농장 2**: 농장 1 Lv.8 완료 시, 총 36별 투자, 2,560코인 수익
- **농장 3**: 농장 2 Lv.8 완료 시, 총 72별 투자, 17,190코인 수익
- **농장 4**: 농장 3 Lv.8 완료 시, 총 108별 투자, 77,120코인 수익

## 데이터 아키텍처

### 데이터베이스
- **서비스**: Cloudflare D1 (SQLite 기반)
- **로컬 개발**: `--local` 플래그로 로컬 SQLite 자동 생성

### 주요 테이블
1. **simulation_sessions**: 시뮬레이션 세션 정보
2. **users**: 사용자 정보 (입장 순서, 화폐 보유량, 하트허용치)
3. **user_farm_progress**: 사용자별 농장 진행 상황
4. **levelup_history**: 레벨업 이력
5. **simulation_stats**: 일별 시뮬레이션 통계
6. **farm_configs**: 농장 설정
7. **level_configs**: 레벨별 조건 및 보상

## 시뮬레이션 로직

### 자동 진행 알고리즘
1. **일별 반복**: 설정된 기간만큼 매일 실행
2. **신규 입장**: 일일 신규 입장자 수만큼 사용자 생성
3. **하트허용치 업데이트**: 모든 기존 사용자 +1
4. **자동 레벨업**: 조건을 만족하는 모든 레벨 자동 진행
5. **별 자동 구매**: 레벨업에 필요한 만큼 자동 구매
6. **통계 저장**: 일별 플랫폼 수익 통계 저장

### 분석 지표
- **사용자별 ROI**: (수익 - 투자) / 투자 × 100
- **플랫폼 수익**: (별 판매액) - (코인 지급액)
- **수익 사용자 비율**: 순이익 > 0인 사용자 수
- **막힌 사용자**: 하트허용치 부족으로 진행 불가

## 사용 방법

### 1. 시뮬레이션 생성
1. 메인 페이지에서 시뮬레이션 이름 입력
2. 일일 신규 입장자 수 설정 (예: 100명)
3. 시뮬레이션 기간 설정 (예: 30일)
4. "생성 및 실행" 클릭

### 2. 결과 분석
1. 생성된 시뮬레이션 카드에서 "분석" 클릭
2. 통계 카드에서 전체 요약 확인
3. 플랫폼 수익 분석 확인
4. 입장 순서별 ROI 차트 확인
5. 일별 플랫폼 수익 차트 확인
6. 사용자별 상세 분석 테이블 확인

### 3. 시나리오 비교
- 서로 다른 일일 입장자 수로 여러 시뮬레이션 생성
- 각 시나리오의 수익성 비교
- 최적의 운영 전략 도출

## 핵심 통찰

### 폰지 구조 분석
1. **초기 입장자 우위**: 하트허용치가 빠르게 증가하여 레벨업 유리
2. **후기 입장자 불리**: 신규 입장자가 계속 필요하여 진행 어려움
3. **플랫폼 수익성**: 초기에는 수익 발생, 후기에는 코인 지급 증가로 수익 감소 가능

### 시뮬레이션 결과 예시
- **100명/일, 30일**: 총 3,000명
- **평균 ROI**: 초기 입장자 +200%, 후기 입장자 -50%
- **플랫폼 수익**: 별 판매액의 약 30-40% (변동 가능)
